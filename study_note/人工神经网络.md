###  首先，什么是“非线性激活”？

在神经网络里，一个神经元的典型计算分两步：

1. **线性加权求和**：z=w1x1+w2x2+...+b
    
2. **激活函数**：a=f(z)
    

如果 f 是一个非线性函数（比如不是 f(z)=z），那它就是**非线性激活函数**。

**简单理解**：线性部分像个“打分”，非线性激活把这个分数进行一个非直线的变换，再输出给下一层。

### 2. 为什么要用非线性？（最关键）

- **如果没有非线性激活**（即 f(z)=z，也叫线性激活），那么无论多少层神经网络，最终能表达的函数依然是**线性函数**（因为线性函数复合后还是线性）。  
    这样的网络能解决的问题非常有限，比如无法解决异或（XOR）问题，也无法拟合曲线。
    
- **加上非线性激活后**，多层网络可以逼近任意复杂的函数（万能近似定理）。非线性让网络有了“弯折”能力，从而产生非线性的决策边界。
    

**一句话**：非线性激活是神经网络**能够学习复杂模式**的根本原因。

**独热向量（One-hot vector）** 是一种将离散的、无序的类别特征转化为数值向量的编码方式。

它的核心思想是：**假设总共有 $N$ 个不同的类别，就构造一个长度为 $N$ 的向量来表示一个样本。这个向量中，只有对应该样本类别的那一个位置为 1，其余位置全部为 0。**

### 📝 举例说明
假设我们有一个“颜色”特征，取值集合为 {红, 绿, 蓝}，那么：

- **红**   → `[1, 0, 0]`
- **绿**   → `[0, 1, 0]`
- **蓝**   → `[0, 0, 1]`

因为这三个向量长度都为3，且每一个单独位置轮流设为“1”，看起来像是“只在某一处发热”，所以叫做“独热”。

### 💡 为什么要用它？
1.  **消除类别间的不当顺序关系**：  
    如果不做独热编码，而直接给类别赋数值 1, 2, 3（标签编码），算法可能会误以为“蓝（3）> 绿（2）> 红（1）”，从而引入错误的距离或大小关系。独热向量将每个类别平摊在一个正交的维度上，任意两个不同类别的向量距离和相似度都相等，很好地避免了这个问题。
2.  **适配需要数值型输入的算法**：  
    逻辑回归、支持向量机（SVM）、神经网络等大部分机器学习模型只能处理数值矩阵，不能直接输入字符串或类别标签，独热编码是连接类别数据与这些模型的常用桥梁。
3.  **与某些降维/表示学习方法的配合**：  
    虽然像 PCA 这样的连续值降维方法一般不直接用于高维稀疏的独热向量（因为可能先需要做中心化等），但在 NLP 中，独热向量是词袋模型（BoW）的基础，可作为神经网络 Embedding 层的输入前身。

### ⚠️ 需要注意的地方
- **维度爆炸**：如果类别数量 $N$ 非常大（例如自然语言处理中的几十万词汇），独热编码会产生超高维且极其稀疏的向量，存储和计算开销巨大。此时通常会采用词嵌入（如 Word2Vec，神经网络中的 Embedding 层）来降维成稠密向量。
- **无法表达类别间的关系**：独热向量假设所有类别完全独立，无法体现“绿”和“蓝”可能比“绿”和“红”更相似这种潜在语义联系。

简单总结，独热向量就是把“是什么”转成“在第几位点亮一盏灯”的 0/1 向量，是机器学习中最基础也最通用的类别特征处理方式之一。

### 梯度下降法要点
#h/aqua **Batch和Epoch**的解释
后向传播算法中每输入一个训练样本就要更新数据，为了简便计算我们输入一批样本（这一批称为一个batch）
### 1. Batch（批次）

**定义**：在一次参数更新（梯度下降）中，**同时处理的样本数量**。

**为什么需要 Batch？**

- 如果一次把所有数据都喂给网络（全批次），计算量巨大，内存可能不够。
    
- 如果一次只处理一个样本（在线学习），梯度方向可能“震荡”很大，收敛不稳定。
    
- Batch 是一种折中：一次处理一小批，计算效率较高，且梯度估计比单样本更稳定。
    

**例子**：

- 数据集有 10000 张图片。
    
- 设定 `batch_size = 128`，则每个批次取 128 张图片，一次更新用这 128 张的平均梯度。
    

**相关术语**：

- **全批次（Full Batch）**：batch_size = 整个数据集。
    
- **小批次（Mini-batch）**：batch_size = 几十到几百（最常见）。
    
- **在线（Online）**：batch_size = 1。
    

---

### 2. Epoch（轮次）

**定义**：**整个训练数据集被完整通过神经网络一次**（即所有样本都被前向传播和反向传播过一次）。

**例子**：

- 数据集 10000 张图，batch_size = 128。
    
- 一个 epoch 需要的迭代次数（iteration）= 10000 / 128 ≈ 79 次。
    
- 训练往往需要多个 epoch（如 50、100、200），让模型反复学习数据。
    

**注意**：

- epoch 数越大，模型有机会看到数据更多次，但过多可能过拟合。
    
- 每个 epoch 结束后通常会将数据打乱（shuffle），避免模型记忆顺序。
    

---

### 3. Channel（通道）

**定义**：输入数据或特征图（feature map）的**某个深度维度**，可以理解为“每个像素位置上的不同特征视图”。

**常见例子**：

- **RGB 图像**：输入 shape = (H, W, 3)，这里的 3 就是通道数（红、绿、蓝）。
    
- **灰度图**：通道数 = 1。
    
- **卷积层输出**：比如某层输出 shape = (batch, H, W, 64)，这里的 64 就是通道数，每个通道代表一种学到的特征（如边缘、纹理等）。
#h/aqua **Channel的数量是说明会有n个卷积核生成相对于的feature map**

```
整个数据集 (10,000 张图)
    │
    ├── Epoch 1
    │      ├── Batch 1 (第 1~128 张) → 更新网络
    │      ├── Batch 2 (第 129~256 张) → 更新网络
    │      └── ...（直到看完全部）
    │
    ├── Epoch 2（数据打乱）
    │      ├── Batch 1' → 更新网络
    │      └── ...
    └── ...
```

## 训练神经网络的建议

![[人工神经网络.pdf#page=16&rect=37,189,548,753|人工神经网络, p.16]]
#### 为什么要进行归一化
1. **加速收敛**  
    神经网络的梯度下降优化过程对特征的尺度非常敏感。如果不同特征的数值范围差异很大（例如一个特征在 [0,1]，另一个在 [0,10000]），那么损失函数的等高线会呈现狭长的椭圆形。梯度下降在这种情况下会来回振荡，导致收敛缓慢。归一化后，等高线更接近圆形，梯度能直接指向最小值，从而大幅加快训练速度。
    
2. **避免梯度消失/爆炸**  
    激活函数（如 Sigmoid、Tanh）在输入值很大或很小时，梯度会趋近于 0。如果输入特征数值过大，经过线性变换后进入激活函数的饱和区，梯度就会非常小，导致网络难以学习（梯度消失）。归一化能将输入控制在激活函数较敏感的区域（如 Sigmoid 的近线性区），缓解这一问题。
    
3. **提高模型稳定性和数值精度**  
    极大或极小的数值在反向传播中累积，可能导致梯度爆炸（数值溢出）或非常微小的梯度被浮点数精度截断。归一化能将这些值保持在合理的数值范围内，使训练过程更稳定。
    
4. **使不同特征具有可比性，提升泛化能力**  
    归一化后，每个特征对损失函数的贡献权重相当，模型不会因为某个特征原始数值大而“偏爱”它。这迫使模型根据特征的实际预测能力而非尺度来学习，通常能提高测试集上的表现（泛化能力）。
    
5. **与正则化、初始化等机制协同工作**  
    很多权重初始化方法（如 Xavier、He 初始化）都假设输入是标准化的（均值为 0，方差为 1）。如果输入不符合这一假设，初始化效果会打折扣。同样，L1/L2 正则化也通常假设特征尺度一致，否则惩罚项对不同特征的约束会不公平。
